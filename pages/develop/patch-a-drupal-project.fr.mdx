# Patch d'un projet Drupal

Les correctifs peuvent être fournis par ou à la communauté pour apporter une correction ou une nouvelle fonctionnalité à un projet Drupal.
Il peut s'agir d'un correctif pour un projet du core ou un projet contribué.

Pour une issue problème Drupal.org, il y a 2 façons d'obtenir un correctif
1. Via un fichier de patch, attaché à l'issue
2. Via une branche git / Merge Request sur GitLab

Dans le premier cas, vous pouvez simplement référencer le fichier patch, localement ou sur Drupal.org via une url absolue.

Dans le second cas, vous devriez toujours le télécharger car le code n'est pas figé.
Pour le télécharger localement, il suffit d'ajouter l'extension `.patch` à l'url. de la Merge Request.

Exemple : 

Pour [cette issue] (https://www.drupal.org/project/drupal/issues/3086883), nous avons
- Un fichier patch dans les premiers commentaires, pour démontrer une preuve de concept, il pourrait être référencé directement dans votre section composer.json https://www.drupal.org/files/issues/2023-03-12/3086883-13.patch.
- Une Merge Request, vous pouvez télécharger le correctif à partir de cette URL https://git.drupalcode.org/project/drupal/-/merge_requests/3642.patch.
