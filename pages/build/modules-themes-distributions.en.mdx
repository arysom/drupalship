# Drupal projects

There are galaxies of Drupal projects, here is a short list of stable modules an themes
that can apply to most use cases + a few distributions.

More projects can be discovered via the [Project Browser](https://www.drupal.org/project/project_browser) initiative.

## Modules

Modules are providing most of the time a single feature. They can work on their own or collaborate with other modules.

### Developer Experience

| Name                                                                     | Description             |
|--------------------------------------------------------------------------|-------------------------|
| [Devel](https://www.drupal.org/project/devel)                            | Enhances most development tasks, exposes them as Drush commands and improves var dumping with libraries like Kint. |
| [Module Builder](https://www.drupal.org/project/module_builder)          | Generates a skeleton or "scaffolding" for a module, along with hints on how to fill them in. Useful for newbie developers to learn how Drupal code works, and seasoned developers who are too lazy to look up what arguments a function has to take. |
| [Stage File Proxy](https://www.drupal.org/project/stage_file_proxy)      | Get production files on a development server on demand instead of rsyncing everything.             |
| [Web Profiler](https://www.drupal.org/project/webprofiler)               | Adds a toolbar at the bottom of every page and shows you all sorts of stats such as the amount of database queries loaded on the page, which services are used and much more. |
| [Backup and Migrate](https://www.drupal.org/project/backup_migrate)      | Baackup and restore your Drupal site. |
| [Config Ignore](https://www.drupal.org/project/config_ignore)            | Ignore certain configuration items from being imported. Typically when config should be regarded as content and not part of the repo. |
| [Examples for developers](https://www.drupal.org/project/examples)       | Examples for Block, Cache, Config and Content Entity, Cron, Database API, Email, Events, Form API, Field, Field Permission, File, Hooks, Javascript, Node Type, Page, Pager, PHPUnit, Plugin Type, Queue, Simple Test, Stream Wrapper, Table Sort, Testing, Tour. |
| [Reroute Email](https://www.drupal.org/project/reroute_email)            | Intercepts all outgoing emails from a Drupal site and reroutes them to a predefined configurable email address. |
| [Event Condition Action](https://www.drupal.org/project/eca)             | Create rules that are triggered by events. |
