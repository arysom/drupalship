# Normalisez vos conventions de nommage

Ceci n'est pas spécifique à Drupal, mais cela s'applique spécifiquement à Drupal en raison de la liberté qu'il autorise.

Drupal fournit un _label_ et un _nom machine_ (par exemple, pour les types de contenu _Article_ et _article_, _Page de base_ et _page_).

Normaliser la façon dont vous nommez les types de contenu, les champs, les vocabulaires, les rôles, les styles d'image, les URL propres... a plusieurs avantages :

*   Améliore la réutilisabilité
*   Améliore la lisibilité
*   Facilite l'intégration du projet pour d'autres collaborateurs

Voici quelques règles qui peuvent être facilement adoptées. Il faut les considérer comme une première idée : lisez la règle puis construisez la vôtre.

## Champs

Les champs sont partout : types de contenu, vocabulaires, utilisateurs, blocs, ... alors commençons par là.

*   Pas de préfixe de type d'entité, les noms de champs doivent rester courts et pourraient être utilisés par un autre type d'entité (par exemple, pour un drapeau, n'utilisez pas document\_status, mais status).  
*   Pas de pluriel, suivant la même règle que les tables SQL (d'ailleurs les champs _sont_ des tables SQL). La multiplicité peut changer avec le temps (par exemple, pas field\_files mais field\_file).  
*   Pour la référence d'entité, utilisez simplement le nom machine de la référence (par exemple, document\_type).  
*   Pour les champs qui sont spécialisés par un module (par exemple, Téléphone ou Lien), utilisez le nom du nom machine du module (telephone, lien).

## Types de contenu

Évitez d'être trop générique... ou trop spécifique.  
Être trop générique donnera une idée vague et pourrait conduire à l'hésitation de l'auteur.  
Être trop spécifique conduira à trop de types de contenu.

Fournissez d'abord une description pour votre type de contenu, le nom viendra de lui-même.  

_Exemple : Événement_

## Rôles

Les rôles doivent être définis à partir du cas d'utilisation du site web. Avoir des personas aidera ici.

Une communauté d'utilisateurs auteurs pourrait être définie par

*   Éditeur : personne pouvant relire et publier du contenu
*   Auteur : personne pouvant rédiger du contenu et l'envoyer à l'éditeur

Un site web immobilier pourrait être défini par

*   Promoteur : chef de projet d'un bâtiment pouvant modifier le contenu des propriétés.
*   Agence : représentant commercial pouvant modifier le statut des ventes pour les propriétés.
